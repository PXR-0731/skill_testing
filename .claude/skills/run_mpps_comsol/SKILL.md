# COMSOL MPPS 声学仿真与数据读取

## 脚本概览

| 脚本路径 | 功能 | 调用时机 |
|---------|------|---------|
| `scripts/comsol_runner.py` | 启动 COMSOL，加载 .mph 模型，注入 .mat 参数，执行仿真，导出 .txt 结果 | 用户提供了模型和参数文件，需要运行仿真时 |
| `scripts/result_reader.py` | 读取 .txt 结果文件，计算统计指标，绘制吸声系数曲线图 | 仿真完成后提取数据画图，或用户直接提供 .txt 文件要求画图时 |

> 详细参数说明、输出格式规范、安装前提和故障排查见 [references/usage-guide.md](references/usage-guide.md)。

## 前提条件

- COMSOL Multiphysics 已安装且许可证有效
- Python 环境已安装 `mph`、`scipy`、`matplotlib`、`numpy`
- 仿真耗时约 10-60s，执行后耐心等待，勿重复调用

## 典型工作流

### 场景 A：仅运行仿真
用户要求运行仿真 → 执行功能一 → 结果 .txt 保存到输出目录

### 场景 B：运行仿真并绘制吸声曲线（最常用）
用户要求仿真并查看吸声性能：
1. 执行功能一，运行仿真
2. 从 stdout 捕获 `SIMULATION_COMPLETE:` 后的 .txt 文件路径
3. 将该路径作为 `--result_file` 参数，执行功能二
4. 向用户展示生成的吸声曲线图片及关键指标（峰值吸声系数、峰值频率、100-5000Hz 平均值）

### 场景 C：读取已有结果并画图
用户直接提供已有的 .txt 结果文件 → 跳过仿真，直接执行功能二 → 展示图片和指标

## 执行规则

- **必须先执行功能一、再执行功能二**（场景 B），功能二依赖功能一生成的 .txt 文件
- 捕获 `SIMULATION_COMPLETE:` 标记后的路径作为功能二的输入，不要推测文件路径
- 若 `comsol_runner.py` 返回非零退出码，停止后续步骤并向用户报告错误
- `result_reader.py` 可独立运行，只要提供有效的 .txt 结果文件
- 仿真耗时 10-60s，等待期间不要重复调用
